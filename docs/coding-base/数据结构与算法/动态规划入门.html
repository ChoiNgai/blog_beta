# 动态规划入门

动态规划分为3个部分：

  1. 确定状态
  2. 转移方程
  3. 初始条件和边界情况

  • 确定状态

    即数列每个元素$f[i]$或矩阵每个元素$f[i][j]$代表着什么

  • 转移方程

    例如斐波那契数列的转移方程是$F(n) = F(n-1)+F(n-2)$，这个表示的是每个元素的关系

  • 初始条件和边界情况

    初始条件，例如$f[0]$等于多少

    边界情况，什么时候停下，$i$的范围是多少

    参考：肝了好多天-动态规划十连-超细腻解析 (opens new window)

# 斐波那契问题

  • 509. 斐波那契数 (opens new window)

$$ F(0) = 0,F(1) = 1 \ F(n) = F(n-1)+F(n-2),n>1 $$

给定$n$，计算$F(n)$

从上述问题可知，初始条件为$F(0)=1,F(1)=1$

方法一（暴力递归-深度优先搜索）：

def fib(self, n: int) -> int:
    if n == 0:
        return 0
    if n == 1:
        return 1
    return self.fib(n - 1) + self.fib(n - 2)

不超时算我输，这里暴力递归显然没有利用上此前已计算出来的结果

方法二（动态规划）：

class Solution:
    def fib(self, n: int) -> int:
        f = [0] * (n+1)	#初始化一个元素个数为n+1且元素全为0的列表
        if n == 0:
            return 0
        if n == 1:
            return 1
        else:
            f[0] = 0
            f[1] = 1
            for i in range(2,len(f)):
                f[i] = f[i-1] + f[i-2]	#转移方程
            return f[-1]	#返回列表最后一个数

上述代码中，$f[i]$表示的是数列中第$i$个元素的数值（确定状态），$f(i) = f(i-1) + f(i-2)$（转移方程），$f(0) = f(1) = 1$（初始状态），迭代到第$n$个元素就停止（边界条件）

上面代码太丑，改一下：

class Solution:
    def fib(self, n: int) -> int:
        
        if n == 0:
            return 0
        else:
            f = [0] * (n+1)	#初始化一个元素个数为n+1且元素全为0的列表
            f[1] = 1
            for i in range(2,n+1):
                f[i] = f[i-1] + f[i-2]	#转移方程
            return f[-1]	#返回列表最后一个数

  • 70. 爬楼梯 (opens new window)

一样先确定状态，然后写出转移方程，最后是初始条件和边界情况

这里$f[i]$的值表示爬楼梯的方法数（确定状态）

因为每次可以爬1或2步，所以到第$i$格楼梯的爬法（个数）都是由到$i-1$和到$i-2$阶楼梯的爬法相加，因而转移方程为$f(i) = f(i-1) + f(i-2)$

初始条件$f(0) = f(1) = 1$（爬第一格楼梯和第二个楼梯只有一种方法），从2开始迭代到n+1（第一个元素用来存储第0格楼梯的爬法了，所以是n+1）从而可以确定边界条件

class Solution:
    def climbStairs(self, n: int) -> int:
        f = [0]*(n+1)	  #生成n+1个元素的列表
        f[0] = f[1] = 1   # 0格楼梯和1格楼梯只需要1步
        for i in range(2,n+1):
            f[i] = f[i-1] + f[i-2]	#转移方程
        return f[-1]	#列表的倒数第一个元素（最后一个元素）

这种是每次可以爬一步或两步的，同理，如果可以爬三步的，那转移方程则为$f(i) = f(i-1) + f(i-2) + f(i-3)$

具体题目看

  • 面试题 08.01. 三步问题 (opens new window)

还是像之前那样先确定状态，然后写出转移方程，最后是初始条件和边界情况

class Solution:
    def waysToStep(self, n: int) -> int:
        if n == 0 or n == 1:
            return 1
        if n == 2:
            return 2
        if n == 3:
            return 4
        else:
            dp = [0] * (n+1)
            dp[0] = dp[1] = 1
            dp[2] = 2
            dp[3] = 4
            for i in range(3,n+1):
                dp[i] = dp[i-1] + dp[i-2] + dp[i-3]
        return dp[-1]%1000000007

上述代码中if部分为特例，else部分显然包括了状态（$f(i)$表示有多少种爬法）；转移方程则变成了$f(i) = f(i-1) + f(i-2) + f(i-3)$，迭代次数为到$n+1$停止（边界条件）

  • 746. 使用最小花费爬楼梯 (opens new window)

之前的题都是有几种爬楼梯的方法，这次题目换了一下，增加的条件是每爬一次都有体力消耗，求爬完楼梯最小体力消耗是多少（体力的最低花费）？

套路：

确定状态：这次$f(i)$表示的不是有几种爬法了，因为题目求体力的最低花费，所以此时$f(i)$表示的是爬到第$i$格楼梯体力的最低花费；

转移方程：$f(i) = min[ f(i-1)+cost(i),f(i-2)+cost(i-1) ]$，$f(i)$表示爬到第$i$格楼梯体力的最低花费，因为每次可以爬楼梯1或2阶，所以爬到第$i$阶有两种可能：

  1. 爬到第$i-1$阶再爬1步到第$i$阶（耗费体力为$f(i-1)+cost[i]$）
  2. 爬到$i-2$阶再爬2步第$i$阶（耗费体力为$f(i-2)+cost[i-1]$）

所以，显然两种方式里哪个小，则哪种就是到第$i$阶的体力最低花费，因而得到转移方程

初始条件和边界情况：（懒得写了）

class Solution:
    def minCostClimbingStairs(self, cost: List[int]) -> int:
        
        n = len(cost)
        if n == 2:
            return min(cost)
        else:
            x = [0] * n
            x[1] = min(cost[0],cost[1])
            for i in range(2,len(x)):
                x[i] = min(x[i-1]+cost[i],x[i-2]+cost[i-1])
            return x[-1]

# 路径问题

  • 62. 不同路径 (opens new window)

机器人位于 m x n 网格的左上角，每次只能向下或者向右移动一步。机器人试图达到网格的右下角

问总共有多少条不同的路径？

还是动态规划，只是一维变二维（即使$f(i)$变$f(i,j)$，莫慌）

还是像之前那样先确定状态，然后写出转移方程，最后是初始条件和边界情况

确定状态:$f(i,j)$表示到第$i$行第$j$列的位置有多少种走法

转移方程:因为只能向下或向右，所以$f(i,j) = f(i-1,j)+f(i,j-1)$

初始条件和边界情况：初始条件为$f(0,0)=1$，注意$0\le i\le m-1$，$0\le j \le n-1$（从零开始数嘛）

class Solution:
    def uniquePaths(self, m: int, n: int) -> int:
        f = [ [0] * n ] * m #生成m*n大小的矩阵
        f[0][0] = 1
        for i in range(0,m):
            for j in range(0,n):
                if i > 0 and j > 0:
                    f[i][j] = f[i-1][j] + f[i][j-1]
                elif i > 0 :
                    f[i][j] = f[i-1][j]
                elif j > 0 :
                    f[i][j] = f[i][j-1]
                else:
                    continue
        return f[m-1][n-1]

# 最大子序和

  • 53. 最大子序和 (opens new window)

最大子序和是经典的动态规划题，之前没做过，但是一看就是老动规题了。 先来看动规三要素：

  1. 确定状态
  2. 转移方程
  3. 初始条件和边界情况

  • 确定状态

    即数列每个元素$f[i]$或矩阵每个元素$f[i][j]$代表着什么，因为这道题因为要求的是最大子序列和，那这里我们将$f[i]$定义为包含了第i个元素的最大子序列和

  • 转移方程 因为我上面已经说了$f[i]$代表的是包括了第$i$个元素的最大子序和： 1、我们假想从第一个元素开始算起，如果第一个元素小于等于0，显然求最终结果的时候不用加上第一个元素（注意，我这里说的是最终结果，因为本方法里的f[0]求的是包含第一个元素的最大子序和）； 2、同理，如果从前面的元素累计和是大于0的，但是加上第i个元素后反而变小了，那么显然不用加上第i个元素，所以这一段求最大子序序列和就此打住。 通过以上分析可以将状态转移方程写成如下：

$$ f[i+1] =\left{ \begin{array}{rcl} f[i]+x[i] & , & {if \quad[i]+x[i] > x[i] }\ x[i] & , & {if \quad[i]+x[i] \le x[i] } \end{array} \right. $$

其中$f[i]$表示包含第i-1个元素前求得的最大子序和（比如f[0]表示的是包含第1个元素的最大子序和，f[1]表示的是包含第2个元素的最大子序和，因为i的范围是[0,n-1] ），$x[i]$表示第i-1个元素，将$max(f[i]+x[i],x[i])$这个结果存储在$f[i+1]$，好让可以利用之前求得的信息 （因为有n个元素，所以f的大小需要设置成n+1，最后一个用于存储最后一步的信息）

  • 初始条件和边界情况

    初始条件，即$f[i]$默认大小是多少，这里设置成无穷小

    边界情况，什么时候停下，$i$的范围是多少

将Python代码写出来是这样的：

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:

        f = (len(nums)+1) * [float(-inf)] #初始状态
        for i in range(len(nums)):		  #边界情况
            f[i+1] = max(f[i]+nums[i],nums[i]) #转移方程
            
        return max(f)
