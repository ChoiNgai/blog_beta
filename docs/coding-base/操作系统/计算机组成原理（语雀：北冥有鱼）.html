# 计算机组成原理

原文链接：https://www.yuque.com/beimingyouyu-9woka/computer/zcrl9i

# 概述篇

# 1 计算机发展历史

# 1.1 计算机发展的四个阶段

第一阶段 1946年——1957年 电子管计算机
第二阶段 1957年——1964年 晶体管计算机
第三阶段 1964年——1980年 集成电路计算机
第四阶段 1980年——至今 超大规模集成电路计算机

# 1.2 微型计算机的发展历史

  1. 单核CPU性能受限

  2. 摩尔定律

  3. 多核CPU

# 2 计算机的分类

  • 超级计算机

  • 大型计算机

  • 迷你计算机（服务器）

  • 工作站

  • 微型计算机（个人计算机）

# 3 计算机的体系与结构

# 3.1 冯诺依曼体系

将程序指令和数据一起存储的计算机设计概念结构

早期计算机仅含固定用途程序，改变程序得更改结构，重新设计电路。而冯诺依曼体系的核心理念是存储程序指令，并设计通用电路。

冯诺依曼体系的计算机必须包含的五个组件：

  • 存储器

  • 控制器

  • 运算器

  • 输入设备

  • 输出设备

CPU = 运算器 + 控制器

冯诺依曼瓶颈：

CPU和存储器速率之间的问题无法调和。

# 3.2 现代计算机的结构

解决CPU与存储设备之间的性能差异。

CPU = 运算器 + 控制器 + 存储器

# 4 计算机的层次与编程语言

  1. 硬件逻辑层

  2. 微程序机器层

  3. 传统机器层

  4. 操作系统层

  5. 汇编语言层

  6. 高级语言层

  7. 应用层

# 5 计算机的计算单位

  • 容量单位
  • 速度单位

网络速度：单位为Mbps，即Mbit/s

CPU速度：CPU的速度一般体现为CPU的时钟频率，单位一般为Hz，即高低电平每秒钟变换的次数。

# 6 计算机的字符与编码集

  • ASCII码：7bits

  • 拓展ASCII码：8bits

  • 中文编码集：GB2312（1980）、GBK（1995）

  • Unicode

# 组成篇

# 1 计算机的总线

例子：USB（Universal Serial Bus）通用串行总线

作用：解决不同设备之间的通信问题。

# 1.1 总线的分类

  1. 片内总线：芯片内部的总线。
  2. 系统总线：

数据总线：双向传输各个部件的数据信息，数据总线的位数（总线宽度）是数据总线的重要参数，一般与CPU位数相同。

地址总线：指定源数据或目的数据在内存中的地址。

控制总线：控制总线是用来发出各种控制信号的传输线。

# 1.2 总线的仲裁

目的：为了解决总线使用权的冲突问题。

仲裁方法：

  1. 链式查询

  2. 计时器定时查询

  3. 独立请求

# 2 计算机的输入输出设备

# 2.1 常见的输入输出设备

常见的输入设备：

  • 字符输入设备：键盘（薄膜键盘、机械键盘、电容键盘）
  • 图像输入设备：鼠标、数位板、扫描仪

常见的输出设备：

  • 图像输出设备：显示器、打印机、投影仪

# 2.2 输入输出接口的通用设计

  1. 数据线：是I/O设备与主机之间进行数据交换的传送线

  2. 状态线：I/O设备状态向主机报告的信号线

  3. 命令线：CPU向设备发送命令的信号线

  4. 设备选择线：主机选择i/O设备进行操作的信号线

# 2.3 CPU与IO设备的通信

  • 程序中断

  • DMA（直接存储器访问）

DMA直接连接主存与IO设备，DMA工作时不需要CPU的参与。

# 3 计算机存储器

# 3.1 存储器的分类

  1. 按存储介质分类
  • 半导体存储器：内存、U盘、固态硬盘
  • 磁存储器：磁带、磁盘
  1. 按存取方式分类

  • 随机存储器（RAM）：随机读取、与位置无关

  • 串行存储器：与位置有关、按顺序查找

  • 只读存储器（ROM）：只读不写

# 3.2 存储器的层次结构

  1. 缓存：CPU寄存器、高速缓存等

  2. 主存：内存

  3. 辅存：硬盘、U盘等

缓存-主存层次

原理：局部性原理

实现：在CPU与主存之间增加一层速度快（容量小）的Cache

目的：解决主存速度不足的问题

主存-辅存层次

原理：局部性原理

实现：主存之外增加辅助存储器（磁盘、SD卡、U盘等）

目的：解决主存容量不足的问题

# 3.3 主存储器——内存

RAM（Random Access Memory）随机存取存储器，通过电容存储数据，必须隔一段时间刷新一次，如果掉电，那么一段时间后将丢失所有数据。

# 3.4 辅助存储器——磁盘

磁盘调度算法：

  • 先来先服务算法

  • 最短寻道时间优先：优先访问离磁头最近的磁道

  • 扫描算法（电梯算法）：每次只往一个方向移动，到达需要服务的尽头再反方向移动

  • 循环扫描算法：始终往一个方向移动

# 3.5 高速缓存

# 3.5.1 高速缓存的工作原理

  • 字：指存放在一个存储单元中的二进制代码组合
  • 字块：存储在连续的存储单元中而被看作是一个单元的一组字

命中率和访问效率是衡量缓存的重要性能指标。

# 3.5.2 高速缓存的替换策略

高速缓存替换的时机：缓存中没有CPU需要的数据，需要从主存载入所需数据。

  • 随机算法

  • 先进先出算法（FIFO）

  • 最不经常使用算法（LFU）

  • 最近最少使用算法（LRU）：一般使用双向链表实现，把当前访问节点置于链表前面，保证链表头部节点是最近使用的。

# 4 计算机的指令系统

# 4.1 机器指令的形式

机器指令的组成：操作码、地址码。

操作码：指明指令所要完成的操作。

地址码直接给出操作数或者操作数的地址。

地址码分类：

  • 三地址指令

地址一中的内容操作地址二中的内容，把结果存放在地址三中。

  • 二地址指令

地址一中的内容操作地址二中的内容，把结果放在地址一中或者地址二中。

  • 一地址指令

地址一中的内容操作完成后，把结果放在地址一中。

  • 零地址指令

通常为空操作、停机操作、中断返回操作等。

# 4.2 机器指令的寻址方式

  • 指令寻址：顺序寻址、跳跃寻址
  • 数据寻址：立即寻址、直接寻址、间接寻址

# 5 计算机的控制器

  1. 程序计数器：用来存储下一条指令的地址

  2. 时序发生器：用于发送时序脉冲，CPU依据不同的时序脉冲有节奏地进行工作

  3. 指令译码器：翻译操作码对应的操作以及控制传输地址码对应的数据

  4. 寄存器

  • 指令寄存器：从主存或高速缓存取计算机指令

  • 主存地址寄存器：保存当前CPU正要访问的内存单元的地址

  • 主存数据寄存器：保存当前CPU正要读或写的主存数据

  • 通用寄存器

  1. 总线

# 6 计算机的运算器

  1. 数据缓冲器

  2. ALU：算术逻辑单元

  3. 通用寄存器

  4. 状态字寄存器

  5. 总线

# 7 计算机指令的执行过程

# 7.1 指令执行过程

  1. 取指令
  • 从缓存取指令（高速缓存）
  • 送到指令寄存器（控制器）
  1. 分析指令

  • 指令译码器译码（控制器）

  • 发出控制信号

  • 程序计数器+1（控制器）

  1. 执行指令

  • 装载数据到寄存器（运算器）

  • ALU处理数据（运算器）

  • 状态字寄存器记录运算状态（运算器）

  • 送出运算结果

# 计算篇

# 1 二进制正负数的表示

1）原码表示法

使用0表示正数，使用1表示负数。规定符号位位于数值第一位。

缺点：0有两种表示方法00和10；原码进行运算非常复杂，特别是两个操作数符号不同的时候。

2）补码表示法

引入补码的目的：减法运算复杂，希望找到使用正数替代负数的方法。

3）反码表示法

目的：找出原码和补码之间的规律，消除转换过程中的减法。

规律：

  • 正数的原码、补码、反码相同

  • 负数的反码等于原码除符号位外按位取反

  • 负数的补码等于反码+1

# 2 定点数与浮点数

# 2.1 定点数

# 2.1.1 定点数的表示方法

​ 小数点固定在某个位置的数称之为定点数。

​ 注：整数部分是零的小数叫做纯小数。

​ 如果不是纯小数或者纯整数，需要乘以比例因子以满足定点数保存格式。

# 2.1.2 定点数的加法运算

数值位与符号位一同运算，并将符号位产生的进位自然丢掉。

判断溢出：双符号位判断法

  • 单符号位表示变成双符号位：0=>00，1=>11

  • 双符号位产生的进位丢弃

  • 结果的双符号位不同则表示溢出

# 2.2 浮点数

# 2.2.1 浮点数的表示方法

引入浮点数的原因：

  • 计算机处理的数很大程度上不是纯小数或纯整数
  • 数据范围很大，定点数难以表达

1）浮点数的表示格式

2）浮点数的表示范围

​ 单精度浮点数：使用4字节，32位来表示浮点数

​ 双精度浮点数：使用8字节，64位来表示浮点数

3）浮点数的规格化

  • 尾数规定使用纯小数
  • 尾数最高位必须是1

# 2.2.2 浮点数的加减法

步骤：

1）对阶

2）尾数求和

3）尾数规格化

4）舍入

5）溢出判断：如果规格化后，阶码双符号位不一致，则认为是溢出

# 2.2.3 浮点数的乘除法

乘法：阶码相加，尾数求积

除法：阶码相减，尾数求商

步骤：

  1. 阶码运算

  2. 尾数运算

  3. 尾数规格化

  4. 舍入

  5. 溢出判断

# 2.3 定点数与浮点数的对比

  • 当定点数与浮点数位数相同时，浮点数表示的范围更大

  • 当浮点数尾数为规格化数时，浮点数的精度更高

  • 浮点数运算包括阶码和尾数，浮点数的运算更为复杂

# 实践篇

# 1 双向链表

单向链表：每一个节点都有下一个节点的地址或引用。

双向链表：每一个节点都有上一个节点和下一个节点的地址或引用。

双向链表的优势：

  • 可以快速找到一个节点的下一个节点

  • 可以快速找到一个节点的上一个节点

  • 可以快速去掉链表中的某一个节点
