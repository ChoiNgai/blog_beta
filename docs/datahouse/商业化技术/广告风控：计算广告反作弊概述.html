# 广告风控：计算广告反作弊概述

# 引言

互联网广告从最早的在线广告到互动广告，再到如今的计算广告,无论是算法、技术、市场模式都经过了一系列的发展或变化[1]^{[1]}[1]。计算广告的兴起使得广告归因、个性化成了现实，因而产业链的角色逐渐演化、计价模式也随之改变，与此同时也让广告网络平台的核心目标从点击量转变为用户量，与之带来的是广告产业的蓬勃发展。但是随着广告经济的增长，计算广告却仍旧有一些广告作弊的问题[2]^{[2]}[2]。

根据White Ops和ANA发布的《 2018-2019 Bot Baseline: Fraud in Digital Advertising 》， 预计2019年全球互联网广告作弊造成的经济损失金额将达到58亿美元[3]^{[3]}[3]。 广告作弊不仅仅对某一方或多方造成经济损失，还造成无效流量并威胁着品牌安全。此外，互联网广告行业也因为广告作弊而经受着信任危机，这种情况并不利于行业的发展。

目前互联网广告主要的形式是CPM、CPC、CPA、CPS[4]^{[4]}[4]，与之对应的各种作弊手段也越来越多。如何解决计算广告中的作弊问题，不仅仅涉及工程技术，还涉及大量的数学模型和计算方法以及社会角色之间博弈。本文将主要以常见的作弊手段、迹象和防作弊办法从工程技术预防角度以及卷积神经网络模型方面来研究计算广告反CPM/CPC/CPA/CPS作弊。

# 作弊角色：

  • **媒体作弊：**制造假流量创造更多的收入。
  • **广告网络平台作弊：**满足广告主对广告代理、DSP质量要求。
  • **广告主竞争对手作弊：**按CPM/CPC结算时，竞争对手通过广告作弊让广告主损失广告费并占用其服务器带宽。
  • **广告主作弊：**为了树立品牌知名度并获得应用商店等平台排名上的优势。
  • **用户作弊：**被强制观看广告或为获得便利享受点击/观看广告后运营商给予的额外服务。

# CPM/CPC

CPM/CPC 广告作弊的思路通常是利用检测规则进行破坏或修改，用低价值或虚假的流量获取广告费。

# 常见的作弊手段及特征

  • 服务器运行脚本自动发起http请求

    将客户端系统、移动设备识别码、mac地址等信息连接成url并以http请求[5]^{[5]}[5]的方式发送给广告检测方，通过脚本[6]^{[6]}[6]不断自动执行。

  • 重复刷新广告

    利用JavaScript脚本[7]^{[7]}[7]重复刷新，使得用户点击一次广告发送多个点击到广告检测方。

  • iframe造假（仅适用于CPC）

    修改iframe[8]^{[8]}[8]属性的width和height将广告缩小至用户不可见，即可让广告运行但又使得用户并没有看到广告。

  • 搜索引擎中反SEO[9]^{[9]}[9]

    网站通过虚假关键词，堆砌关键词获得搜索排名的前列或显示给通过关键词搜索本不应搜索到的用户，或设置隐形页面、文本来干扰影响广告的匹配。

  • 肉鸡

    被控制的用户在不知情的情况下程序运行使得浏览或点击广告。

  • Root

    用户终端系统获取超级管理员权限后程序后台运行浏览或点击广告。

  • 诱骗用户点击

    作弊手段：前端界面伪装成下载等诱骗用户点击。

  • 饱和式变现

    程序后台浏览或点击广告，用户并没有观看。

# 防治措施

  • 屏蔽主要的云服务商ip段

    用户不会租聘云主机去浏览广告，并且云服务器提供厂商ip通常是按段，因而可以坚定地将云服务器提供厂商的ip段封锁。此法针对使用云服务器运行脚本自动发起http请求的作弊。

  • 快速傅里叶变换分析频域

    此法针对使用脚本重复刷新广告的作弊。

# CPA/CPS

# 常见的作弊手段及特征

  • 频繁切换身份

    频繁切换身份避免web端的cookie、苹果端IDFA、安卓端 AndroidID的验证。

  • 流量劫持

    正常情况是当用户点击广告主的广告时， 检测程序在广告网络平台的埋点给用户加上Cookie，以此归因最终计算广告费。部分广告网络平台强行加上cookie信息，使得不管用户是否从该广告网络平台最种都归因为是从该广告网络平台点击。

  • Cookie Stuffing

    在广告投放过程中，屏蔽掉对广告市场起监管者如搜索引擎、主要广告网络平台的IP地址，不对其展示广告。 导致相关监管人员投诉或举报 ， 因而使监管人员的审查成本增加并能延长广告的投放时间 。

  • 强行归因

    对所有用户都记录下载或绑定上记录了解来源的cookie

  • 广告主自行刷

    游戏运营商往自己的游戏充值或网站应用自发提高下载量以获取更高的排名和更多的流量。

  • 网店代销

    充当广告网络平台一方的代理直接从广告主接手广告投放任务，并在另一流量更大的店铺销售商品，用户下单即直接从广告主的店铺发货。

# 防治措施

  • 以SDK方式潜入媒体SSP产品，获取终端的信息

    以SDK方式潜入媒体SSP产品，获取终端的信息对比电量、屏幕尺寸等是否同一终端。

  • 加密数据，http协议改用https协议

    网站增加SSL加密传输协议，使得不太安全的http改成较为安全的https协议

  • 广告召回、定向受众、CTR预估

  • 终端设备活跃度分析

    将用户系统授权的可用信息通过分类出正常使用情况的移动设备和一直用于广告作弊的移动终端。

  • 图文点击验证

    让广告展示或归因时增加用户图文验证。
