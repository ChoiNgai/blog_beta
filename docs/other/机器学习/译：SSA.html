# 樽海鞘群算法：工程设计问题的仿生优化器（SSA）

【关键词】：粒子群优化、多目标优化、遗传算法、启发式算法、算法、基准

【摘要】

这项工作提出了两种新颖的优化算法，称为樽海鞘群算法（SSA）和多目标樽海鞘群算法（MSSA），用于解决具有单个和多个目标的优化问题。SSA和MSSA的主要灵感来自于樽海鞘群在海洋中航行和觅食时的行为。这两种算法在几种数学优化函数上进行了测试，以观察和确认其有效行为，从而找到优化问题的最佳解决方案。数学函数的结果表明，SSA算法能够有效地改善初始随机解，并趋于最优。MSSA的结果表明，该算法可以逼近Pareto最优解，具有较高的收敛性和覆盖率。本文还考虑使用SSA和MSSA解决一些具有挑战性和计算量大的工程设计问题（例如，机翼设计和船用螺旋桨设计）。实际案例研究的结果证明了所提出的算法在解决具有困难和未知搜索空间的现实世界问题中的优点。

# 介绍

在过去的十年中，元启发式技术 出乎意料地受欢迎。这种流行是由于以下几个主要原因：灵活性，无梯度机制以及这些算法的局部最优避免。前两个优势源自以下事实：元启发法仅通过查看输入和输出来考虑和解决优化问题。换句话说，元启发式方法将优化问题假设为黑匣子。因此，不需要计算搜索空间的导数。这使得它们非常灵活，可以解决各种问题。由于元启发式算法属于随机优化技术家族，因此它们来自随机运算符。这有助于他们在解决实际问题时避免使用局部解决方案，而实际问题通常具有大量的局部最优值。

元启发式算法分为两大类：进化[1]和群体智能[2]技术。进化算法模仿自然进化的概念。此类中最好和最受推崇的算法是遗传算法（GA）[3]。该算法模拟达尔文进化论的概念。在GA中，针对特定问题的优化是从一组随机解开始的。通过目标函数评估解决方案后，它会根据其适应度值修改解决方案的变量。由于最好的人有更大的机会参与改进其他解决方案，因此随机初始解决方案很有可能得到改善。文献中还有其他几种进化算法，例如差异进化（DE）[4]，进化策略（ES）[5]和进化规划（EP）[6,7]，以及基于生物地理的优化（BBO） ）算法[8]。

群智能技术模仿了自然界中的成群动物，牧群，学校或鸡群的智力。这些算法的主要基础源自一组生物的集体行为。例如，蚂蚁可以集体地保证殖民地的生存，而无需集中的控制单元。换句话说，没有人告诉蚂蚁在哪里和如何找到原始食物，但他们合作找到离巢室甚至很远的食物。此类中两个最受欢迎的算法是蚁群优化（ACO）[9]和粒子群优化（PSO）[10]。ACO算法模仿了蚂蚁的社会行为，以寻找巢穴和原始食物之间的最短路径。PSO算法模拟了鸟类在导航和狩猎中的集体行为。文献中的其他群体智能技术包括：人工蜂群（ABC）算法[11]，布谷鸟搜索（CS）算法[12]，萤火虫算法（FA）[13]，蝙蝠算法（BA）[14]，灰狼优化（GWO）[15–17]，海豚回声（DE）[18]，鲸鱼优化算法（WOA）[19]，果蝇优化算法（FOA）[20]以及 和声搜索[21,22]。

不管进化智能技术和群智能技术之间的区别如何，共同点是在优化过程中改进了一个或一组解决方案。如果一种算法仅改善一种解决方案，则称为个人主义算法。如果一组解决方案得到了改进，则称为集体算法。个性化算法是有益的，因为所需功能评估的次数很少，而且整个优化过程也很简单。但是，陷入局部最优的可能性非常高。集体算法能够更好地避免局部解决方案，并交换有关搜索空间的信息。但是，这样的技术需要更多的功能评估。一些个人主义算法是禁忌搜索（TS）[6,23]，爬山[24]，迭代局部搜索（ILS）[25]和模拟退火（SA）[26]，变邻域搜索（VNS）[27]和导向局部搜索[28]。众所周知的集体算法是GA，ACO，PSO，DE和ES。

尽管在文献中提出算法的优点，但 没有免费的午餐（NFL）[29]证明这些算法都不能解决所有优化问题。换句话说，在解决所有优化问题时，所有元启发式算法的性能都相似。该定理揭示了新算法和特定算法在不同领域中的重要性，因为算法在解决一系列问题中的有效性并不能保证其在不同测试问题中的成功。这是本文的动机，其中首先提出了一种新的元启发式优化算法来解决单目标问题，然后将其扩展到多目标版本。本文的其余部分安排如下。

第二部分回顾了文献和相关著作。第3节介绍了提出的灵感和数学模型。本节还提出了樽海鞘群算法（SSA）和多目标樽海鞘群算法（MSSA）。在第4节中介绍和讨论了两种算法在各种基准函数上的定性和定量结果。在第5节中，SSA和MSSA都被用来解决一些具有挑战性的实际问题。最后，第6节总结了工作并提出了一些未来的研究方向。

变量，目标，变量范围和约束的集合创建了搜索空间/景观。该搜索空间存在于d维空间中，其中d是变量数。对于1D，2D和3D问题，我们可以轻松地在笛卡尔坐标系中绘制搜索空间并观察其形状。但是，绘制大于3的尺寸是不可能的，因为它们超出了我们每天遇到的尺寸。因此，解决优化问题时，第一个挑战是大量变量。

变量的范围限制了搜索空间并且是变化的。变量本身可以是连续的或离散的，它们在其中创建连续的或离散的搜索空间。在以前的情况下，搜索空间中每两个点之间有无限个点。但是，在后一种情况下，两个点之间有一组有限的点。在一个连续的空间中找到全局最优值与在一个离散的空间中找到最优值是不同的，每个最优方法都有其自身的挑战。尽管大多数优化问题都带有变量范围，但在优化过程中仍存在一些未考虑特定范围的问题。一个例子是训练神经网络（NN）的问题[30]。。连接权重和偏差可以是任何实数。解决此类问题也需要特殊考虑。例如，优化器可能从初始范围开始，然后在优化过程中对其进行扩展。

约束条件进一步限制了搜索空间。它们在搜索空间中造成了空白，因为这些区域中的解决方案不适合该问题。例如，螺旋桨叶片的厚度由于易碎性而不能低于一定数值。一组约束甚至可以将搜索空间划分到不同的分离区域。违反约束区域的解决方案称为不可行解决方案。相反，约束区域内的解称为可行解。在文献中，对于搜索空间在受约束区域之内和之外的部分有两个术语：可行区域和不可行区域。受约束的搜索空间尽管在不受约束的搜索空间中具有良好的性能，却有可能使算法无效。一些实际问题，例如计算流体动力学问题，已经占据了不可行的地区。因此，优化技术应配备合适的运算符[31]也处理约束。

解决优化问题时的另一个挑战是本地解决方案的存在。变量，目标函数和约束创建的搜索空间可能非常简单或复杂。在大多数文献中，局部解的数量被认为是优化算法的主要难点。在单目标搜索空间中，有一个最佳解决方案（所谓的全局最优）可以返回最佳目标值。但是，通常还有许多其他解决方案，使返回值接近全局最优目标值。这种解决方案称为局部解决方案，因为如果考虑到附近的搜索空间，它们是局部最好的解决方案，但是在考虑全局范围时，它们并不是全局的最佳解决方案。局部解决方案的存在导致许多优化算法陷入困境。局部最优停顿是指一种情况，其中优化算法找到了局部解，并且错误地将其视为全局最优。实际的搜索空间通常具有大量的本地解决方案。因此，优化算法应该能够有效地避免它们来确定全局最优。

在解决优化问题时，收敛速度也是一个难题。能够避免局部求解的算法不一定能够收敛于全局最优。当算法避免局部求解时，便找到了全局最优解的粗略位置。下一步是提高所获得的粗糙解的准确性。文献中的术语“收敛”是指算法向全局最优值的速率或行为。当然，快速收敛会导致局部最优停滞。相比之下，解决方案的突然变化会导致避免局部最优，但会降低收敛速度，朝着全局最优方向发展。这两个折衷是算法在解决实际问题时面临的主要挑战。

解决实际的单目标搜索空间时，还有其他困难，例如：欺骗性最优，最优隔离，不确定性，嘈杂的目标函数，动态目标函数，可靠的最优等等。这些困难中的每一个都需要特别考虑。这些概念超出了本文的范围，因此有兴趣的读者可以参考Boussaid [32]进行的调查。

# 单目标优化算法

文献中的优化算法可分为两大类：确定性与随机性。如果确定性算法从相同的起点开始，则它们总是会为给定的问题找到相同的解决方案。这种方法的主要优点是可靠性，因为它们可以在每次运行中确定一个解决方案。但是，局部最优停滞是一个缺点，因为这些算法在解决优化问题时通常不具有随机行为。第二类优化算法，即随机方法，是从随机算子中受益的。即使起点保持不变，这也会导致找到不同的解决方案，因此，与确定性方法相比，随机算法的可靠性较低。但是，随机行为会促进局部最优避免，这是随机算法的主要优势。细调和增加运行次数可以提高随机方法的可靠性。

随机优化算法分为两类：个人主义和集体主义。在第一个系列中，随机算法启动并使用单个解决方案执行优化。该解决方案可以随机更改和改进，以达到预定的步骤数或满足最终标准。该系列中最受人尊敬的算法是TS [6,23]，爬山[24]，ILS [25]和SA [26]。该组的优点是计算成本低，并且功能评估次数少。

但是，集体技术会创建多个随机解，并在优化过程中对其进行演化。解决方案的集合通常可以协作以更好地确定搜索空间中的全局最优值。该领域最著名的算法是：GA，PSO，ACO和DE。多种解决方案减少了局部最优停滞的机会，这是集体算法的主要优势。但是，每个解决方案都需要一个功能评估，并且在解决方案之间建立有效的协作具有挑战性。尽管有这两个缺点，但是如今，集体随机优化技术非常流行。这种方法的应用可以在科学和工业中找到。主要原因之一是避免本地解决方案的高能力。

如上所述，实际搜索空间具有大量的本地解决方案。因此，解决方案和随机成分的收集都有助于为实际问题找到更好的解决方案。集体算法的另一个优点是灵活性。这样的技术能够共同克服不同类型的搜索空间中的挑战。也不需要梯度信息，因为随机集体算法将问题视为黑匣子，仅监视问题的输入，输出和约束。最后但并非最不重要的一点是，简单是另一个优势，因为大多数集体算法都模仿群，群，学校或背包中的简单规则。

不管集体随机算法之间的差异如何，它们都遵循相同的过程来近似全局最优。首先从一组随机解开始优化，然后要求它们进行组合，随机，快速和突然地变化。这导致解决方案在全球范围内移动。此过程称为搜索空间探索，因为解决方案由于突然的变化而趋向于搜索空间的不同区域。探索阶段的主要目标是确定搜索空间中有希望的区域，并避免使用局部解决方案。经过足够的探索，解决方案开始逐渐变化并局部移动。此过程称为开发，主要目的是提高准确性

探索阶段获得的最佳解决方案。尽管在开发阶段可能会发生局部最佳避免，但是搜索空间的覆盖范围并不像探索阶段那样广泛。在这种情况下，解决方案避免在全局最优值附近使用局部解决方案。

上段表明，勘探和开发阶段寻求冲突的目标。这里的问题是什么时候最好的时间是从勘探过渡到开采。由于集合算法的随机性以及搜索空间的未知形状，因此没有人能够回答这个问题。因此，大多数算法都平滑地要求搜索代理使用自适应机制从探索阶段过渡到开发阶段[36]。

由于随机集合算法的优势，在该领域提出新算法的兴趣日益浓厚。根据灵感可以将它们分为四个主要类别：群体灵感，物理灵感，进化和基于人类的灵感。最受群体启发的算法是PSO，ACO，ABC

# 多目标优化算法

解决多目标问题的方法主要有两种：先验与后验[48]。在前一种方法中，通过汇总目标将多目标问题转换为单目标问题。这是通过一组通常由专家定义的权重来完成的，并规定了每个目标的重要性。目标汇总后，可以使用单目标优化程序来找到最佳解决方案。该技术的主要缺点是可以通过重新运行算法并更改权重来构造Pareto最优集和前沿。另外，由于这种方法中加法权重的性质，无法确定Pareto最优前沿的凹面区域[49-51]。。关于前一个问题，文献中已有一些改进，但仍需要多次运行才能找到一个以上的Pareto最优解[52]。

但是，后验方法保持了多目标

问题的提法。主要优点是可以一次运行确定Pareto最优集[53]。同样，任何类型的Pareto最优阵线都可以近似，并且没有专家可以定义的权重。通过这种方法，会有相比，决策者许多不同的解决方案，一个革命制度党，ORI方法。另一方面，主要缺点是需要解决多目标和特殊机制以确定Pareto最优集和前锋。这使得后验优化更具挑战性，计算成本也更高。由于与先验方法相比，后验优化的优点不仅在于缺点，因此，本文的重点是后验多目标优化。

随机优化文献中的多目标优化技术已经配备了新的机制，可以找到真实Pareto最优解和前沿具有均匀分布的精确近似值。该领域最受欢迎的算法之一是基于分解的多目标进化算法（MOEA / D）[54]。该算法利用聚合方法从给定的条件创建子问题（未翻译）

问题。每个子问题都是一个单目标问题，但是MOEA / D与纯聚合方法之间的主要区别在于，通过使用其他相邻子问题的信息来优化子问题。换句话说，MOEA / D使用进化算子来组合和演化子问题。该算法有一些改进和变种，可以在[35]中找到。

基于DE的方法[55,56]在文献中也很流行。为了设计DE的多目标版本，对复制操作符进行了广泛的修改。例如，在Pareto边界微分进化（PDE）[57，58]中，从非支配解的集合中选择双亲，以进行交叉。在基于PSO的技术中，找到非主导解决方案和更新非主导解决方案很容易，但是要选择其中一个作为最佳解决方案（例如，在PSO中是gbest或pbest），并存储分布最广的解决方案就具有挑战性[59,60]。最近在这一领域已经完成了许多工作。流行的方法是：使用轮盘选择[59,61,62]，小生[63–65]和sigma方法[66]。

无疑，最受欢迎的多目标算法是非支配排序遗传算法版本2（NSGA-II）[67,68]。该算法根据个人的级别对他们进行排序。这意味着非主导解决方案被分配为等级1，由排名第一的解决方案主导的个人被分配为等级2，依此类推。然后在排名的个人之间进行选择和交叉。等级数越低，产生新种群的选择和参与的可能性就越高。NSGA-II还配备了一个新的固定算子，以保持在所有目标中获得的非支配解决方案的高分布。

显然，模拟樽海鞘数学模型是不能直接用于解决优化问题。换句话说，需要对模型进行一些微调以使其适用于优化问题。单目标优化器的最终目标是确定全局最优。在SSA群模型中，跟随者樽海鞘跟随着领导者樽海鞘。领导者樽海鞘也朝着食物来源方向移动。因此，如果将食物源替换为全局最优值，则樽海鞘链会自动向其移动。然而，问题在于优化问题的全局最优是未知的。在这种情况下，假定到目前为止所获得的最佳解决方案是全局最优方案，并被认为是樽海鞘链将要追寻的食物来源。

  • SSA算法会保存到目前为止获得的最佳解决方案，并将其分配给食物来源变量，因此即使整个群体恶化，它也不会丢失。
  • SSA算法仅更新相对于食物来源领先的樽海鞘的位置，这是迄今为止获得的最佳解决方案，因此领导者始终在探索和利用其周围的空间。

  • 参数c 1在迭代过程中会自适应降低，因此SSA算法首先探索搜索空间，然后加以利用。
  • SSA算法只有一个主要控制参数（c 1）。
  • SSA算法简单易实现。

# 多目标Salp Swarm算法（MSSA）

如第2节所述，多目标问题的解决方案是一组称为帕累托最优集的解决方案。SSA算法能够将樽海鞘推向食物源，并在迭代过程中对其进行更新。但是，由于以下两个原因，该算法无法解决多目标问题：

  • SSA仅将一个解决方案保存为最佳解决方案，因此无法将多个解决方案存储为针对多目标问题的最佳解决方案。
  • SSA使用到目前为止在每次迭代中获得的最佳解决方案来更新食物来源，但是对于多目标问题没有唯一的最佳解决方案。

第一个问题可以通过为SSA算法配备食物源库来解决。该存储库保持了迄今为止在优化过程中获得的最佳非支配解，并且与多目标粒子群优化（MOPSO）中的归档非常相似。该存储库具有最大尺寸，可以存储有限数量的非支配解决方案。在优化过程中，使用Pareto优势运算符将每个樽海鞘与所有存储库樽海鞘进行比较。如果樽海鞘主导了存储库中的一个解决方案，则必须交换它们。如果樽海鞘主导了存储库中的一组解决方案，则应将所有解决方案从存储库中删除，并将该樽海鞘添加到存储库中。如果至少有一个存储库樽海鞘在新种群中主导了一个樽海鞘，则应立即将其丢弃。如果相对于所有存储库樽海鞘而言，该樽海鞘并没有被占主导地位，则必须将其添加到归档中。

这些规则可以确保存储库始终存储该算法到目前为止获得的非支配解。但是，在特殊情况下，存储库已满，并且与存储库居民相比，樽海鞘占主导地位。当然，最简单的方法是随机删除存档中的一种解决方案，并用主导的樽海鞘替换它。明智的方法是删除存储库中类似的非主导解决方案之一。由于后验多目标算法应该能够找到均匀分布的帕累托最优解，因此从归档中删除的最佳候选者是在分布稠密地区。这种方法改善了迭代过程中储存库居民的分布。

# 使用MSSA的船用螺旋桨设计

本小节中研究的问题是另一个CFD问题。实际上，船用螺旋桨的形状已得到优化。螺旋桨设计问题的主要两个目标是：最大化效率和最小化气蚀[122]。船舶的发动机旋转连接到螺旋桨的轴。轴使螺旋桨旋转，从而为车辆创造信任。效率是指最终转化为推力的发动机功率。然而，由于电机中存在高密度的水和摩擦，因此无法实现无能量损失的推力[123]。

除推力外，螺旋桨还会产生气蚀[124]。螺旋桨在水中旋转时，会高速旋转水。加速水流并使其通过叶片会降低水压。这导致形成气泡，该气泡崩溃并引起强烈的局部冲击波。这些微小的冲击波侵蚀了螺旋桨的表面并缩短了其使用寿命。空化是不希望的，优化器最好找到空化程度最低的设计。

螺旋桨的形状在确定效率和空化方面起着重要作用。螺旋桨由多个类似的叶片组成，这些叶片围绕着轴分组。为了设计螺旋桨，螺旋桨中有许多结构参数：叶片的数量，叶片的长度，轴的厚度，轴的长度和叶片的形状。毫无疑问，叶片的形状是定义效率和空化的最重要组成部分。
